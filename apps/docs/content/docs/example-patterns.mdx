## Which pattern should you choose?

<Accordions type="single">
  <Accordion title="I already have a PDF on disk">
    Start with the Node.js file-path example. It is the simplest call shape and
    works well for backend jobs or batch pipelines.
  </Accordion>
  <Accordion title="My PDF arrives over HTTP or from object storage">
    Use the in-memory buffer pattern so you can keep everything off disk.
  </Accordion>
  <Accordion title="I need to expose this behind a web UI">
    Use a server route or route handler. Keep the native package out of client
    bundles.
  </Accordion>
  <Accordion title="I need smaller preview payloads">
    Keep PNG as the default when quality matters most. Use WebP or JPEG when
    you want to shrink response sizes for previews or downstream transport.
  </Accordion>
  <Accordion title="I want to send pages into another system">
    Convert first, then pass the rendered page images into whatever comes next,
    whether that is OCR, an image API, a multimodal model, or your own
    processing pipeline.
  </Accordion>
</Accordions>

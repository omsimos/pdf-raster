---
title: Quickstart
description: Convert PDFs into PNG pages from a file path or an in-memory buffer.
---

`convert()` is the only public entrypoint you need to learn first.

## Convert all pages from a file path

```ts
import { convert } from "@omsimos/pdf-to-images";

const pages = await convert("./report.pdf");

for (const page of pages) {
  console.log({
    pageIndex: page.pageIndex,
    width: page.width,
    height: page.height,
    dpi: page.dpi,
    mimeType: page.mimeType,
  });
}
```

<Callout type="info" title="Default behavior">
  If `pages` is omitted, the full document is rendered.
</Callout>

## Convert selected pages

Page indices are zero-based:

```ts
import { convert } from "@omsimos/pdf-to-images";

const pages = await convert("./report.pdf", {
  pages: [0, 2],
  dpi: 300,
});
```

## Understand the result

Every page includes:

- `pageIndex`
- `data` as a PNG `Buffer`
- `mimeType`
- `width`
- `height`
- `dpi`

These values are enough to:

- save the image
- display it in a UI
- upload it to another API
- feed it into downstream processing

## Common first mistakes

<Accordions type="single">
  <Accordion title="Why are page numbers zero-based?">
    The API uses JavaScript-style zero-based indices because `pages` is an
    array of page indices, not human-readable page labels.
  </Accordion>
  <Accordion title="Why does the library return PNG buffers instead of file paths?">
    Returning bytes keeps the package flexible for APIs, workers, storage, and
    other downstream consumers without forcing filesystem writes.
  </Accordion>
  <Accordion title="Should I call this in a browser component?">
    No. Keep it in server code only.
  </Accordion>
</Accordions>

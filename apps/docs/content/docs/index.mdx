<Callout type="info" title="Single responsibility">
  `@omsimos/pdf-to-images` converts PDF pages into PNG images with a single
  native API.
</Callout>

`@omsimos/pdf-to-images` is a native server-side library for turning PDFs into
PNG page images quickly.

It fits cleanly anywhere you need page images instead of PDF bytes:

<Steps>
  <Step>
    Start with a PDF from disk, object storage, an upload, or another API.
  </Step>
  <Step>
    Use `@omsimos/pdf-to-images` to render the document into PNG page images.
  </Step>
  <Step>
    Use the PNG output in previews, background jobs, OCR flows, VLM prompts,
    or any other downstream image workflow.
  </Step>
</Steps>

## What the library does

<Cards>
  <Card title="One public API" href="/docs/api-reference">
    The package centers on `convert(input, options?)`.
  </Card>
  <Card title="Native rendering" href="/docs/runtime-and-environment">
    Rendering is handled through Rust, `napi-rs`, and PDFium.
  </Card>
  <Card title="Lossless output" href="/docs/quickstart">
    Each page comes back as a PNG `Buffer` with dimensions and DPI metadata.
  </Card>
</Cards>

## What it intentionally does not do

- no browser runtime support
- no JPEG or image-encoding choice in v1
- no document editing, annotation, or extraction APIs

That narrow API keeps the package easy to reason about. You can use the PNG
output for previews, storage, background jobs, OCR, VLM prompts, or other
downstream processing without changing how the conversion layer works.

## Output shape

Each page is returned as:

```ts
type ConvertedPage = {
  pageIndex: number;
  data: Buffer;
  mimeType: "image/png";
  width: number;
  height: number;
  dpi: number;
};
```

## Why this package exists

Many systems do not want PDF bytes directly. They want page images that are:

- lossless
- easy to inspect
- high enough resolution for reading and analysis
- straightforward to pass into downstream APIs

This package makes that conversion step small, explicit, and fast.
